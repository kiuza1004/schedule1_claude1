# 제품 요구사항 문서 (PRD)
## Schedule Manager - 개발 일정 및 프롬프트 관리 앱

**작성일**: 2026-04-18  
**버전**: 1.0.0  
**상태**: 활성

---

## 1. 비즈니스 배경

### 문제 정의
- 개발 중인 여러 기능들의 정보가 텍스트 파일로 흩어져 있음
- 언제 어떤 기능을 개발했는지 추적 불가
- 작성한 프롬프트를 재사용하기 어려움
- 개발 진행도를 한눈에 파악하기 어려움

### 솔루션
- 통합 웹 애플리케이션을 통한 모든 개발 정보 중앙화
- 캘린더 기반의 직관적 일정 관리
- 강력한 검색 기능으로 빠른 조회
- 체계적인 CRUD 작업

## 2. 목표 및 성과 지표

### 주요 목표
1. 개발 정보의 체계적 관리

## 3. 기능 명세서

### 3.1 핵심 기능 (MVP)

#### 3.1.1 달력 뷰
```
요구사항:
- 월간 달력 표시
- 해당 날짜의 개발 항목 개수 표시
- 날짜 클릭 시 그날의 모든 항목 표시
- 이전/다음 월 이동

UI/UX:
- 반응형 디자인 (모바일/태블릿/데스크톱)
- 명확한 색상 구분 (계획/진행중/완료)
- 스무스 애니메이션
```

#### 3.1.2 개발 항목 등록 (Create)
```
입력 필드:
1. 개발 타이틀 (필수)
   - 타입: Text
   - 최대 길이: 100자
   - 예: "사용자 인증 시스템"

2. 개발 내용 (필수)
   - 타입: Textarea
   - 최대 길이: 1000자
   - 플레이스홀더: "상세한 기능 설명을 입력하세요"

3. 프롬프트 (필수)
   - 타입: Textarea
   - 최대 길이: 5000자
   - 플레이스홀더: "AI 모델에 제공할 프롬프트를 입력하세요"

4. 개발 시기 (필수)
   - 타입: Date Picker
   - 기본값: 오늘 날짜
   - 범위: 과거 1년 ~ 미래 1년

5. 상태 (필수)
   - 타입: Select/Radio
   - 옵션: 계획 / 진행중 / 완료
   - 기본값: 계획

6. 우선순위 (선택)
   - 타입: Select
   - 옵션: 낮음 / 중간 / 높음
   - 기본값: 중간

검증 규칙:
- 모든 필수 필드 입력 확인
- 타이틀 중복 체크 (동일 날짜)
- 프롬프트 길이 체크

제출 후:
- 성공 메시지 표시
- 폼 초기화
- 달력에 항목 추가
```

#### 3.1.3 개발 항목 조회 (Read)

**3.1.3.1 목록 뷰**
```
표시 항목:
- 타이틀
- 개발 시기
- 상태 (badge)
- 우선순위 (아이콘)
- 수정/삭제 버튼

정렬 옵션:
- 날짜순 (최신순/오래된순)
- 우선순위순
- 상태순

페이지네이션:
- 한 페이지 10개 항목
- 다음/이전 버튼
```

**3.1.3.2 상세 뷰**
```
표시 내용:
- 타이틀
- 개발 내용 (전체)
- 프롬프트 (전체)
- 개발 시기
- 상태
- 우선순위
- 생성일시
- 수정일시

액션:
- 복사 버튼 (프롬프트 클립보드 복사)
- 수정 버튼
- 삭제 버튼
```

#### 3.1.4 개발 항목 수정 (Update)
```
동작:
1. 기존 항목 정보 로드
2. 수정 폼 표시
3. 사용자 입력 받음
4. 검증
5. 확인 팝업 표시 (필수!)

확인 팝업:
- 메시지: "정말로 수정하시겠습니까?"
- 변경 전/후 비교 보기
- 확인/취소 버튼

수정 완료 후:
- 성공 메시지
- 목록으로 돌아가기
```

#### 3.1.5 개발 항목 삭제 (Delete)
```
동작:
1. 삭제 버튼 클릭
2. 최종 확인 팝업 표시 (필수!)

확인 팝업:
- 메시지: "정말로 삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다."
- 타이틀 표시
- 확인/취소 버튼
- 1초 딜레이 후 확인 버튼 활성화 (실수 방지)

삭제 완료 후:
- 성공 메시지
- 항목 제거
- 달력 업데이트
```

### 3.2 검색 기능

#### 3.2.1 일자별 검색
```
입력:
- 시작 날짜 선택
- 종료 날짜 선택 (선택사항)

결과:
- 해당 기간의 모든 개발 항목 표시
- 날짜순 정렬
- 해당 항목 수 표시

예시:
- 2026-04-01 선택 → 04-01의 모든 항목 표시
- 2026-04-01 ~ 2026-04-30 선택 → 4월 모든 항목 표시
```

#### 3.2.2 키워드 검색
```
검색 범위:
- 타이틀
- 개발 내용
- 프롬프트

옵션:
- 대소문자 구분 안함
- 부분 검색 지원
- 태그 검색 (선택 기능)

결과:
- 매칭 항목 표시
- 키워드 하이라이트
- 검색 결과 수 표시

예시:
- "인증" 검색 → "사용자 인증", "JWT 인증" 등 모두 표시
```

#### 3.2.3 고급 필터
```
필터 옵션:
1. 상태 필터 (체크박스)
   - ☐ 계획
   - ☐ 진행중
   - ☐ 완료

2. 우선순위 필터 (체크박스)
   - ☐ 낮음
   - ☐ 중간
   - ☐ 높음

3. 조합 검색
   - 키워드 + 상태
   - 날짜 범위 + 우선순위
   - 모든 필터 조합 가능

결과:
- 실시간 업데이트
- 필터 리셋 버튼
- 검색 결과 내보내기 (CSV)
```

## 6. API 엔드포인트 명세

### 6.1 일정 관련 API

| 메서드 | 엔드포인트 | 설명 |
|--------|-----------|------|
| POST | `/api/schedules` | 새 일정 생성 |
| GET | `/api/schedules` | 모든 일정 조회 (페이지네이션) |
| GET | `/api/schedules/:id` | 특정 일정 조회 |
| GET | `/api/schedules/search` | 일정 검색 (키워드, 날짜 등) |
| PUT | `/api/schedules/:id` | 일정 수정 |
| DELETE | `/api/schedules/:id` | 일정 삭제 |
| GET | `/api/schedules/date/:date` | 특정 날짜의 일정 조회 |
| GET | `/api/schedules/stats` | 일정 통계 |
